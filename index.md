# Introduction

This project designs and implements a relational database for a simulated Pokémon TCG card shop. The system covers the full lifecycle of a card shop: inventory, customers, orders, deliveries and payments.

The database implements automated logic via triggers, transactional business logic via stored procedures, and reusable query abstractions via views. A minimal Python service exposes database functionality, and pgvector enables semantic card search as the NoSQL component.

The project is documented here on GitHub Pages. All sprint work, stories, decisions and outcomes are recorded in this documentation so that any reader can follow the full progression without requiring access to Jira.
